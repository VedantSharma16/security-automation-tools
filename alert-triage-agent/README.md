# Alert Triage Agent

A small, well-tested log triage engine that sits between raw logs and an
analyst's attention: it matches log lines against a MITRE ATT&CK-mapped
ruleset, then produces a prioritized triage report — either deterministically
(rule-based) or as an LLM-generated narrative (via Claude).

Built to show both sides of the same problem: the detection-engineering
mindset (rules, severities, ATT&CK mapping, precise regex matching) and the
AI-engineering mindset (a pluggable summarization layer, prompt design for
a specific analyst task, graceful degradation when no API key is present).

## Why this exists

Most "SOC in a box" toy scripts either grep for a hardcoded keyword list, or
wrap an LLM around raw logs and hope for the best. Neither is how real
detection pipelines work. This project keeps detection logic deterministic,
auditable, and testable (`rules/default_rules.yaml`), and treats the LLM as
an optional reporting layer on top of that — not a replacement for it.

## Architecture

```
logfile ──▶ parser.py ──▶ rules.py (RuleEngine) ──▶ summarizer.py ──▶ TriageReport
            LogEvent[]      Finding[]                 Summarizer
                                                        ├── RuleBasedSummarizer  (default, offline)
                                                        └── AnthropicSummarizer  (--llm, needs ANTHROPIC_API_KEY)
```

- **`models.py`** — `LogEvent`, `Rule`, `Finding`, `TriageReport`, and an
  ordered `Severity` enum.
- **`parser.py`** — normalizes raw text into `LogEvent`s (line number +
  source, for traceability back to the original file).
- **`rules.py`** — loads rules from YAML, validates them (unique IDs, valid
  regex, valid severity) at load time rather than at match time, and scans
  events in a single pass.
- **`summarizer.py`** — a `Summarizer` protocol with two implementations:
  - `RuleBasedSummarizer`: deterministic breakdown by severity and ATT&CK
    technique, plus the top findings. No network calls, no API key. This is
    the default and what the test suite exercises end-to-end.
  - `AnthropicSummarizer`: sends the top findings (already filtered and
    ranked, not the raw log) to Claude with a SOC-analyst prompt, and
    returns an executive summary, likely attack narrative, and prioritized
    next actions. Only imported when `--llm` is passed, and fails with a
    clear error if the `anthropic` package or `ANTHROPIC_API_KEY` is missing.
- **`pipeline.py`** — wires the above together and renders a report to
  dict/JSON.
- **`cli.py`** — `triage-agent scan <logfile>` with `--rules`, `--llm`,
  `--model`, `--format {text,json}`, `-o/--output`.

## Detection rules

Rules live in `rules/default_rules.yaml`, not in code, so the ruleset can
grow without touching the engine. Each rule declares:

```yaml
- id: TOOL-001
  name: Credential dumping tool referenced
  pattern: '(mimikatz|secretsdump|hashdump|dumpsec|lsass\.exe.*dump)'
  severity: critical
  mitre_technique: "T1003 - OS Credential Dumping"
  description: Process or command line referencing a known credential-dumping tool or technique.
```

The bundled ruleset ships 18 rules mapped to MITRE ATT&CK, covering SSH brute forcing, account
creation, credential dumping (Mimikatz/secretsdump), C2 frameworks (Cobalt
Strike, Metasploit, Rubeus, Impacket), lateral movement (PsExec,
CrackMapExec), encoded PowerShell, scheduled-task persistence, event-log
clearing, LOLBin downloads (certutil, bitsadmin), and reverse-shell netcat
usage — see the YAML file for the full list and rationale.

## Usage

```bash
pip install -e ".[dev]"        # core + test deps
pip install -e ".[ai]"         # optional: adds `anthropic` for --llm

# Deterministic triage report (no API key required)
triage-agent scan sample_logs/auth_sample.log

# JSON output, for feeding into another tool/dashboard
triage-agent scan sample_logs/auth_sample.log --format json -o report.json

# LLM-narrated triage report (requires ANTHROPIC_API_KEY)
export ANTHROPIC_API_KEY=sk-...
triage-agent scan sample_logs/auth_sample.log --llm

# Bring your own ruleset
triage-agent scan /var/log/auth.log --rules my_rules.yaml
```

## Testing

```bash
pip install -e ".[dev]"
pytest -q
```

25 tests cover the parser, rule-loading/validation edge cases (duplicate
IDs, invalid regex, invalid severity), the full pipeline against the bundled
sample log, JSON serialization, and the CLI (including error paths for a
missing log file and a malformed rules file). The `AnthropicSummarizer` path
is tested for its fail-closed behavior (raises clearly without an API key)
rather than mocked network calls, so the suite runs fully offline.
