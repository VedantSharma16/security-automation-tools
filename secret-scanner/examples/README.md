# examples/

A small, deliberately vulnerable sample repo used to demo Secret Scanner
end-to-end, generated by [`generate_demo.py`](generate_demo.py) rather than
committed as static files.

## Why generated instead of committed

Every "secret" in the demo is synthetic — either a randomly generated
string with no real-world meaning, or a well-known documented placeholder
(the AWS access key ID is literally the example value from
[AWS's own docs](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html)).
Nothing here is a live credential.

That doesn't matter to a pattern-based secret scanner, though — including
GitHub's own push protection, which every push to this repository goes
through. It matches on *format*, not on whether a value actually works, so
a committed file containing a format-plausible fake Stripe/Slack/AWS key
gets blocked exactly like a real leaked one would. Rather than working
around that (which would mean disabling a real security control), the demo
repo is generated locally at run time, and `generate_demo.py`'s own source
builds every value from separate string fragments so it doesn't read as a
contiguous secret either. It's a small illustration of the same
format-vs-validity distinction this tool itself is built around.

## Usage

```bash
python examples/generate_demo.py
python -m secretscanner.cli examples/vulnerable_repo
```

Expected output: 14 findings spanning an AWS access key ID + secret access
key, a Google API key, a Stripe key, a JWT, a GitHub token, an npm token, a
Slack bot token and webhook, a Twilio key, a hardcoded Postgres connection
string, a fake RSA private key block, and one generic high-entropy secret
plus one generic high-entropy password (caught by entropy scoring, not a
vendor signature). The `service_account_key = "your_api_key_here"` line in
the generated `config/settings.py` is intentionally **not** flagged — it
demonstrates the placeholder filter that keeps entropy-based rules from
alerting on obvious dummy values.

```bash
# Gate CI on high/critical only, ignore medium-severity generic findings:
python -m secretscanner.cli examples/vulnerable_repo --fail-on high
```
